## 実装・バグ改修のワークフロー

実装（機能追加）・バグ改修を行うときは、**必ず最初に `implement` スキル
（`.claude/skills/implement/SKILL.md`）を読み、そのワークフローに従うこと。**

概要（詳細はスキル本文を参照）:

1. 最新の `main` から新しくブランチを切る
2. 実装後に `npm test` を実行し、全件パスさせる
3. `npm run e2e` を実行する（スクリーンショットは `screenshots/` に出力される）
4. ユーザーに動作確認・承認を求める
5. 承認後に `main` へ向けて PR を作成する
